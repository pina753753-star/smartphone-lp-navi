# 納品タイプ・画像差し替え前提のLP制作ナビ設計

## 方針

このツールは、フォーム回答後にプロンプトだけを出すのではなく、API連携版ではその場でデザインプレビューを出すことを前提にする。

ユーザー体験は以下にする。

1. フォームに回答する
2. 納品先・納品タイプを選ぶ
3. 画像素材あり/なしを選ぶ
4. AIがLP構成・文章・画像設計・デザイン案を生成する
5. その場でデザインサンプルを表示する
6. 写真・商品画像・実績画像だけ差し替えられる状態にする
7. 納品形式に合わせて出力する

## 納品先

- ココナラ
- BASE
- STORES
- その他

## 納品タイプ

- 画像LP納品
- HTML/CSS ZIP納品
- GitHub Pages公開納品
- 商品ページ素材セット
- デジタル商品ZIP
- Canva編集用構成納品
- ココナラ納品セット
- 外部LP＋購入ボタン案内

## 画像素材の扱い

### 画像ありの場合

依頼者の写真・商品画像・ロゴ・口コミ画像・実績画像を使う前提で、デザインサンプルを生成する。

ただし、画像はコードやデザイン内で差し替えやすいように以下のような枠に分ける。

- hero-main.jpg
- product-01.jpg
- product-02.jpg
- customer-voice-01.jpg
- customer-voice-02.jpg
- before-after-01.jpg
- mockup-screen-01.jpg

### 画像なしの場合

画像なしでも成立するように、以下をHTML/CSSで作る。

- 写真差し替え枠
- スマホモック枠
- 実績カード
- 口コミカード
- Before/Afterカード
- CTAカード

### AI画像生成ありの場合

先に以下の画像を生成する。

- ファーストビュー用ヒーロー画像
- スマホモック内のサンプル画面
- 商品紹介用ビジュアル
- 実績・口コミ用の写真枠
- CTA周りの装飾
- セクション装飾

## デザインプレビューで必ず出すもの

- ファーストビュー
- 強いCTA
- スマホモック
- 3つのメリットバッジ
- 悩み共感ブロック
- Before/After
- 商品紹介ブロック
- 実績数字カード
- 口コミカード
- FAQ
- 最終CTA

## ココナラ向け注意

- 外部連絡を前提にしない
- ZIP納品・正式な納品を前提にする
- 権利範囲を明記する
- 修正回数を明記する
- Google Drive納品前提にしない

## BASE向け注意

- 商品ページ素材セットを基本にする
- 商品内容がわかる説明文を出す
- 専用商品でも内容がわかるようにする
- デジタル商品ZIPは1GB以下を想定する
- スマホだけでHTML編集できる前提にしない

## STORES向け注意

- STORES内にHTML/CSSを直接入れる前提にしない
- 商品画像・バナー・説明文・FAQを基本にする
- 外部LP＋STORES BUTTON案内を用意する
- ファイル名は半角英数字・ハイフン・アンダーバーにする

## 表現チェック

美容・健康・サプリ・ダイエット・エステ系は以下を避ける。

- 必ず痩せる
- 100％効果
- 治る
- 若返る
- No.1根拠なし
- 医師推奨風の断定
- ビフォーアフターの誇大表現
- 口コミの捏造
- 架空実績
