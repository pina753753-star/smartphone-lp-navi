# スマホLP制作ナビ

スマホだけでCodexを使ってLPを作りたい初心者向けの、GitHub Pages対応ツールです。LP制作に必要な情報を入力すると、LP構成案、セクション別文章案、デザイン指示、画像生成プロンプト、Codex用完成プロンプト、GitHub公開手順、エラー時の修正プロンプトを自動生成します。

## 概要

- 外部ライブラリなしで動く静的サイトです。
- `index.html`、`style.css`、`script.js`、`README.md`だけでGitHub Pagesに公開できます。
- 背景画像にはルート直下の `tool-bg.PNG` を使います。
- 文字、入力欄、選択欄、ボタン、生成結果はHTML/CSSで表示します。
- 入力内容はブラウザのlocalStorageに保存されます。
- スマホ幅390pxで崩れにくく、PCでは中央にスマホ幅で表示されます。

## 使い方

1. `index.html` をブラウザで開きます。
2. 「LP設計をはじめる」を押してフォームへ移動します。
3. 依頼者名、商品名、ターゲット、悩み、CTAなどを入力します。
4. 「LP制作指示書を作る」を押します。
5. 生成された各カードの「コピー」ボタンで必要な項目をコピーします。
6. まとめて使いたい場合は「全部まとめてコピー」を押します。
7. 入力を消したい場合は「リセット」を押します。

未入力の項目があっても生成できます。未入力の内容は「未入力」として扱われるため、あとから編集できます。

## GitHub Pagesで公開する方法

1. GitHubで新しいリポジトリを作成します。
2. このリポジトリの以下のファイルをアップロードします。
   - `index.html`
   - `style.css`
   - `script.js`
   - `README.md`
   - `tool-bg.PNG`
3. リポジトリの `Settings` を開きます。
4. 左メニューから `Pages` を開きます。
5. `Build and deployment` の `Source` を `Deploy from a branch` にします。
6. `Branch` を `main`、フォルダを `/root` にして保存します。
7. 数分待って、表示された公開URLを開きます。

### 反映されない時の確認

- `index.html` がリポジトリのルート直下にあるか確認してください。
- `tool-bg.PNG` のファイル名がHTML/CSS内の指定と完全に一致しているか確認してください。大文字小文字も区別されます。
- GitHub Pagesの設定が `main / root` になっているか確認してください。
- GitHubの `Actions` にエラーが出ていないか確認してください。
- ブラウザのキャッシュを削除するか、別のブラウザで確認してください。

## 背景画像の差し替え方法

このツールは `style.css` の `.storybook` で `tool-bg.PNG` を背景画像として読み込んでいます。

```css
background-image: linear-gradient(rgba(245, 250, 255, 0.16), rgba(255, 250, 240, 0.22)), url("tool-bg.PNG");
```

差し替える場合は、次のどちらかで対応してください。

- 新しい画像を `tool-bg.PNG` という名前でルート直下に置き換える。
- 別のファイル名にする場合は、`style.css` の `url("tool-bg.PNG")` を実際のファイル名に変更する。

`images` フォルダに入れる場合は、例として `url("images/tool-bg.PNG")` のように変更してください。

## 注意点

- 画像内に日本語文字を入れると崩れることがあるため、文字はHTML/CSSまたはCanvaなどで後から入れる前提です。
- コピー機能はHTTPSまたはlocalhostで安定して動きます。通常のGitHub Pages公開URLでは問題なく使えます。
- localStorageは同じブラウザ内だけに保存されます。別端末や別ブラウザには引き継がれません。
- CTAリンク先URLは生成結果に反映されますが、公開前に必ず実際にタップして確認してください。
- 横スクロールが出る場合は、長いURLや英数字が原因のことがあります。CSSでは折り返し設定を入れていますが、公開前にスマホで確認してください。
